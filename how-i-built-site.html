How I built this site using Bootstrap

12 February 2013

My original blog was built in Drupal, a widely-used CMS, mainly because Drupal is what I'm most familiar with. However, the platform is a bit of an over-kill for a simple website. I'd heard a lot of good things about Bootstrap, built by the people who created Twitter.

This is how I went about it:

  1. Create a folder structure with /css and /js subfolders. This is your working folder.
  2. Download and unzip Bootstrap source from this page. It's the second button. This will contain several folders and sub-folders, including docs/assets/css and docs/assets/js.
  3. My site is based on the Fluid layout on this page, so copy docs/examples/fluid.html from your unzipped folder to your working folder. Rename it index.html.
  4. Copy all the .js files from docs/assets/js to /js in your working folder. Copy bootstrap.css to /css in your working folder.
  5. Now fix the paths to the /css and /js folders and check that you page looks like the Fluid layout on the Bootstrap examples page.
  6. Now start customising your page. I didn't want a login box at the right of the nav bar, or a lefthand column, so I removed these, replacing 'span9' with 'span12' in the code. I also added some css at the top of some pages, to customise them a bit more.
  7. Add Twitter and email icons using FontAwsome icons. These are highly customisable.

You now have a working blog. This is the point where you can customise it. There are two stages to this - one is to select only those components that you need, the other is to alter the colours to your liking. It doesn't really matter if you select more items than you need, it just makes the css and js files larger, which will slow down your site a bit. Make a note of the settings, especially the colours, in case you need to alter these. If you think you'll add more functionality to your site later, such as an image gallery, add the required components now.

  • 1. Choose components:
    • ScaffoldingSelect all the items
    • ComponentsSelect Navs, Navbar, Media component
    • MiscellaneousNone
    • Base CSSHeadings, Code, Buttons and Icons
    • ResponsiveAll the items
    • JS ComponentsCollapse and Carousel
  • 2. Select jQuery plugins:
    • Collapse and Carousel.
  • 3. Customize variables:
    • @purple#5d3d42
    • @pink#d3a3a2
    • @textColor@purple
    • @navbarBackground@purple
    • @navbarBackgroundHighlightlighten(@navbarBackground, 10)
    • @navbarBrandColor@white
    • @navbarLinkColor@pink
    • @navbarLinkColorHover@white
    • @navbarLinkColorActive@white

I needed to make a minor change to css/bootstrap-responsive.css. On line 1007 or thereabouts, background-color: #f2f2f2; was replaced by background-color: #ab8483; The minified version of the file is only 5K smaller than the unminified version, so I did not bother to minify it again.

Note:If you are using the Chrome browser and have the Hootlet extension installed, you will see any words preceded by '@' and any numbers preceded by '#' as links. To fix this, right-click the Hootlet icon, select Options and toggle Link Twitter status and user profile handles into the HootSuite dashboard to Off. Thank you Page 1/2 Full for posting this workaround.

All my customised css was put into one file - css/blog.css.

I added a .htaccess file to enable friendly urls, ie. about.html is the same as about.

Finally, I checked the HTML in each page. The easiest way to do this is via the Web Developer toolbar/Tools in the Chrome browser. Checking the CSS throws up a lot of errors in bootstrap.css. This is due to the way the w3c validator works. These errors, according to esteemed sources, can be ignored. Note: add 508 check when url points to this

Bootstrap Pros

  • Very easy to set up. If you want to use it straight out of the box, you don't need much technical knowledge beyond basic HTML and CSS.
  • No server-side coding is involved. It runs directly in the browser, so you don't need to set up a local server for testing.
  • There are tools for customisation. You'd need to experiment with these to get the effects you want.

Bootstrap Cons

  • Out of the box, there are limited options.
  • It's a bit fiddly to customize. In particular, there seems to be no way of storing the customized variables in 3. above.

Conclusion

For me personally, I'd use components of Bootstrap in other projects. For example, the Forms plugin, with it's built-in validation looks really nice.
